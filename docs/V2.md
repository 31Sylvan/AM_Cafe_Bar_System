# Coffee Shop OS V2

V2 聚焦真实门店日常运营效率：减少查账成本、降低初始化门槛、让关键单据可追溯。

## 已落地

- 列表筛选
  - 采购单：按日期、状态筛选
  - 销售单：按日期、状态筛选
  - 盘点单：按日期、状态筛选
  - 库存流水：按日期、流水类型筛选
- 单据详情
  - 采购单详情：单头、采购明细、数量、单价、金额
  - 销售单详情：单头、销售明细、理论成本、理论毛利
  - 盘点单详情：理论库存、实际库存、盘盈盘亏差异
- 数据初始化模板
  - 原料模板
  - 产品模板
  - 员工模板
  - 采购明细模板
- 商品导出表接入
  - 页面：`/imports/products`
  - 支持上传门店商品导出 `.xls/.xlsx`
  - 提取商品名、原平台分类、系统分类、售价、上下架状态、售卖渠道、商品简介和图片
  - 自动跳过非标准经营商品、重复名称、空名称和无售价商品
  - 支持确认导入产品表，按 `store_id + name` 更新或插入
  - 为真实订单导入建立产品档案基础，减少订单商品未匹配
- CSV 导入预检
  - 上传 CSV 后校验必填列、枚举值、数字格式、日期格式和重复名称
  - 当前阶段只返回预检报告，不直接写入数据库
  - 已支持销售批量录入模板，适配每日渠道汇总数据的预检
- 真实订单 Excel 接入
  - 页面：`/imports/orders`
  - 支持上传店内订单和自提订单 `.xls/.xlsx`
  - 自动过滤未支付、已取消订单
  - 自动拆分 `商品信息`，生成标准销售批量 CSV
  - 商品别名映射页面：`/products/aliases`
  - 支持把订单商品名映射到系统产品，例如把外部订单里的促销名、套餐名或渠道命名映射到标准产品
  - 支持确认导入销售单，按外部订单号幂等去重
  - 正式导入前会先按系统产品名匹配，再按商品别名匹配；发现未匹配商品时停止写入
  - 正式导入前会校验订单涉及产品是否已维护配方；缺配方时停止写入，避免销售入账但库存不扣减
  - 正式导入时按系统产品售价做折扣比例分摊，并调用数据库 RPC 完成销售单、销售明细、库存扣减和现金流水
  - 本地 Demo 模式只做导入校验模拟，不写入真实数据库
- 补货建议
  - 按安全库存、当前库存和最近 30 天消耗估算建议采购量
  - 输出建议预算、预计可用天数和优先级
  - 支持 CSV 导出
- 月结流程
  - 通过 `month_close_snapshots` 固化指定月份的利润、成本和现金余额
  - 同一门店同一月份可重新生成，便于上线初期校准口径
  - 支持 CSV 导出和备份中心归档
- 移动端盘点模式
  - 大字号卡片式录入
  - 单手输入实际库存，底部固定提交按钮

## V2 状态

代码侧 V2 任务已完成。真实上线前仍需执行最新 Supabase migration，并用真实门店数据跑一次验收。
